HK Legal Chatbot: Full-stack RAG app for parsing, indexing, and querying Hong Kong legal documents (e-Legislation PDFs).

## Commands (agent-safe, non-interactive)

```bash
python -m venv .venv
.venv\\Scripts\\python -m pip install -r backend/requirements.txt
```

```bash
.venv\\Scripts\\python backend/main.py
```

## Code Style & Conventions

### Backend (Python)

Formatting:
- 4-space indentation; prefer double quotes.
- Keep changes consistent with surrounding files; avoid reformatting unrelated code.

Imports:
- Order: stdlib → third-party → local.
- Local imports are absolute from `backend.*` (not relative-dot imports).

Types:
- Use type hints on public functions/methods and FastAPI models.
- Prefer `typing` imports (`Optional`, `Dict`, `List`, `Any`) consistent with existing code.

Naming:
- `snake_case` for functions/vars/modules.
- `PascalCase` for classes (e.g., `PDFLegalParserV2`, `VectorStoreManager`).
- `UPPER_SNAKE` for constants.
- Prefix private helpers with `_`.

Error handling & logging:
- Existing pattern is `try/except Exception as e:` with `print(...)` and safe fallbacks.
- Do not swallow errors with `except: pass`.
- When appropriate, include context tags in prints (e.g., `[EmbeddingService] ...`).

Environment + GPU/DLL setup (critical):
- `backend/core/setup_env.py` is used to set up CUDA/DLLs.
- **Must call** `setup_env.setup_cuda_dlls()` before importing torch/onnxruntime in modules that use them.

Paths:
- Use `os.path` with paths relative to `__file__` for committed code.
- Absolute paths are acceptable only inside `if __name__ == "__main__":` blocks for local dev.

FastAPI + schemas:
- Use Pydantic models for request/response bodies (see `backend/main.py`).

### Frontend (JavaScript / React)

Formatting:
- 2-space indentation; prefer single quotes.
- Keep diffs minimal; do not introduce a new formatter.

Components:
- Functional components with hooks only (`useState`, `useEffect`).
- Props destructuring is the norm.

Naming:
- Components/files: `PascalCase` (e.g., `ChatInterface.jsx`).
- Variables/functions/props/state: `camelCase`.

Styling:
- Tailwind utility classes only.
- Dark mode is class-based (`dark:` variants + toggling `document.documentElement.classList`).
- Do not add per-component CSS files; keep global tweaks in `src/index.css`.

API usage:
- Use `fetch()` directly to `http://localhost:8000`.
- Streaming responses are handled via `ReadableStream` reader in existing chat UI.

Lint rules (source of truth):
- `frontend/eslint.config.js` (ESLint 9 flat config).
- `no-unused-vars` ignores names matching `^[A-Z_]`.
